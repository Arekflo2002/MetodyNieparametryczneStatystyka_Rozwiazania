Funkcja obliczania mocy testu¶

Najoglniejsza funkcja, ktra przygotuje dane, ktre wykorzystam, aby zrobi symulacje w funkcji symulacja. Wic ustalamy liczb danych oraz stopie swobody i idziemy dalej przekazujc parametry dalej do funkcji. Po symulacji dodajemy wynik do tablicy i idziemy dalej.

Argumenty:

  • test - Mwi ktry test jest symulowany

Funkcja bdzie zwraca tablice 2D(i,j), ktra zbieraa dane dla i-tej liczbie danych oraz j-tych stopni swobody.

def obliczanie_mocy_testu(test):
    # Tablica, ktra zbierze mi moce testw, ktre zwrci
    moce_testow = []

    # Dla kadej liczby z liczby_danych
    for l_danych in liczby_danych:
        """
        Tworz now tablic ktra przechowuj wyniki mocu testu dla i-tej liczby danych. Lista przechowuje wyniki dla kadego j-stopnia
        swoobody w stopnie_swobody. Wykorzystuje List Comprahension, eby umilic kod
        """
        moce_testow.append([symulacja(l_danych,s_stopien,test) for s_stopien in stopnie_swobody])

    # Zwracam w.w tablice
    return moce_testow

Funkcja symulacji¶

Ustalilimy ju liczb danych oraz liczbe stopni swobody dla naszych danych. W tej funkcji bdziemy generowa dane z rozkadu t-studenta dokadnie z tymi parametrami, przekazanymi jako argumenty. Nastpnie przelemy je do funkcji, ktra wybierze, ktry test jest obecnie symulowany i wynik tej funkcji bdziemy sumowa w zmiennej suma_odrzucen_H0.

Argumenty:

  • l_danych - Ile danych bdziemy losowa z rozkadu t-studenta
  • s_stopien - Liczba stopni swobody dla rozkadu t-studenta
  • test - Jaki test jest obecnie symulowany

Funkcja bdzie zwraca odsetek w jakim odrzuciem hipotez zerow dla podanych parametrw

def symulacja(l_danych,s_stopien,test):
    # W tej zmiennej bd sumowa ile razy odrzuciem hipotez zerow dla danego zestawu 
    suma_odrzucenia_H0 = 0

    # Tutaj bdzie symulacja
    for k in range(liczba_symulacji):
        # Tworz prbk danych z rozkadu t-Studenta o l_danych wielkoci oraz s_stopien stopni swobody
        probka = stats.chi2.rvs(s_stopien,size=l_danych)

        # Nastpnie przekazuje wszystkie dane, aby wybrac ktry test jest symulowany i dowiedzie si jaki jest wynik testu
        suma_odrzucenia_H0 += wybor_testu(probka,test,s_stopien)

    # Zwracam odsetek odrzucenia hipotezy zerowej
    return suma_odrzucenia_H0/liczba_symulacji

Funkcja wyboru testu¶

Funkcja za argumenty przyjmuje wszystkie dane, ktre s jej potrzebne do wybrania oraz przeprowadzenia danego testu. Argumenty, to:

  • test - Jaki test ma zosta wykonany
  • probka - Zestandaryzowany zestaw danych, ktre bdziemy bada

Funkcja bdzie zwraca 1 lub 0 w zalenoci od wyniku testu(1-odrzucamy test, 0-nie ma podstaw)

def wartosci_dla_chi2(probka,s_stopien):
    """
    Jest to funkcja ktra zwrci 2 zbiory: zbior zaobserwowanych wartosci(na podstawie probki przekazanej do 
    funkcji) oraz zbior wartosci oczekiwanych potrzebnych do testu Chi kwadrat. Zaczne od wylosowania probki 
    z rozkadu chi2 z parametrami takimy jakimi standaryzowaem oryginalna probke. Nastepnie podziele je 
    na przedzialy zawierajce czestotliwosci wystepowania dane. Na koniec wyrwnam rznice w sumie tych tablic,
    aby uniknac pozniejszych bledow.    
    """
    # Losuje probke z rozkl normalnego o parametrach z proby
    probka_chi2_or = stats.chi2.rvs(size = len(probka),df = s_stopien)

    # Dzielenie danych na przedzialy
    probka_chi2_or, przedzialy = np.histogram(probka_chi2_or,bins='auto')
    
    # Teraz tam gdzie przedzial ma czestotliwosc 0(co prowadzi do wysypania sie testu chi kwadrat) cze zewnetrzne
    # przedzialy w jedno
    indexes = np.nonzero(probka_chi2_or==0)[0]
    probka_chi2_or = probka_chi2_or[probka_chi2_or != 0]
    przedzialy = np.delete(przedzialy,indexes)

    #Teraz dziele na przedzialy probke pochodzaca z chi2 oryginalnej
    probka, _ = np.histogram(probka,bins=przedzialy)

    # Teraz wyrownuje roznice w sumach tych tablic dajac obserwacje ktore sie nie zmiescily w tym zasiego do ostatnich 
    # przedzialow 
    roznica = sum(probka_chi2_or) - sum(probka)
    probka[0] += math.ceil(roznica/2)
    probka[-1] += math.floor(roznica/2)
    
    # Zwracam tablice gotowe do testu
    return probka,probka_chi2_or


def wybor_testu(probka,test,s_stopien):

    """
    Oglna zasada dziaania testw:

    Jeeli pvalue z danego testu jest mniejsze ni wczesniej ustalony poziom istotnotnosci to odrucamy 
    Hipoteze zerowa co znaczy ze zwracamy warto 1. W przeciwnym wypadku zwracamy 0
    
    """

    # Najpierw podejmiemy wybr jaki test mamy wykona

    # Test Kolgomorova
    if test == "kol":
        if stats.kstest(probka,'chi2',args=(s_stopien,)).pvalue < poziom_istotnosci: return 1

    # Test Chi kwadrat
    elif test == "chi":
        fobs,fexp = wartosci_dla_chi2(probka,s_stopien)
        if stats.chisquare(fobs,f_exp=fexp).pvalue < poziom_istotnosci: return 1
    
    # Jeeli do tej pory nie zwrcono 1 to znaczy, e aden z testw nie odrzuci H0 wic moemy zwrcic 0
    return 0

Wnioski¶

Wraz ze wzrostem liczby danych wzrasta take moc testu, aczkolwiek wraz ze wzrostem stopni liczby swobody w rozkadzie t-studenta moc testu spada i dy do 0. Z wykresu widzimy, e moc testu maleje bardzo szybko i ju dla 10 stopni swobody jest bliski 0. Wydaje si, e test jest miarodajny i w miar wiarygodny tylko dla:

  • duej liczby stopni swobody(powyej 100)
  • maej liczby stopni swobody, ale duej liczbie danych(powyej 50)

Wynika to z faktu, e Test Kogomorowa, jest bardziej oglny ni test Shapiro-Wilka.

Jak widzimy na wykresie test Shapiro-Wilka jest najbardziej monotonicznym testem w tym zestawie. Natomiast test Komogorowa idzie w jedn skrajno, jak jest bardzo szybki spadek i denie do 0.
